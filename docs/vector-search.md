# Векторный поиск — план реализации и тестирования

## Идея

Текущий BM25-поиск работает по точным словам: "акула" находит посты где написано "акула", но не находит посты про "нападение в море" или "опасный обитатель".

Векторный поиск превращает текст в числовой вектор (эмбеддинг) так, что близкие по смыслу тексты дают близкие векторы. Запрос "как провести день на яхте" найдёт пост "утро начинается с подъёма паруса" даже без совпадения слов.

## Архитектура

### Хранение эмбеддингов

```
data/
  posts/
    seapinta.json          # посты как сейчас
  embeddings/
    seapinta.json          # массив { id, vector }
```

Эмбеддинг рассчитывается от поля `textClean + ' ' + (ocrText || '')`.

### Модель

`text-embedding-3-small` (OpenAI)
- 1536 измерений
- ~$0.02 за 1M токенов
- Стоимость для 11k постов: ~$0.10

### Алгоритм поиска

1. Расширяем запрос синонимами через OpenAI (уже есть)
2. Превращаем запрос в вектор через Embeddings API
3. Считаем cosine similarity запроса с каждым постом
4. Объединяем BM25-скор и vector-скор (гибридный поиск)
5. Возвращаем top-K

### Гибридная формула

```
finalScore = alpha * bm25Score + (1 - alpha) * vectorScore
```

`alpha = 0.3` — по умолчанию (BM25 хорош для точных имён, vector — для смысла).


## План тестирования

### Подход

Тестируем поиск по 10 реальным темам из контент-плана. Для каждой темы заранее определены **релевантные посты** (ground truth). Метрика — попадает ли хотя бы один релевантный пост в top-5 результатов.

### Темы и ожидаемые результаты

| # | Запрос | Ожидаемые посты (минимум 1 из списка в top-5) |
|---|--------|----------------------------------------------|
| 1 | Почему яхтенные путешествия — один из лучших форматов отдыха | `silavetrasila_5403`, `silavetrasila_3938`, `meetingplace_news_176` |
| 2 | До конца предпродажи осталось 6 дней | `regataveka_35`, `silavetrasila_8156`, `silavetrasila_8184` |
| 3 | Как проходит один день на яхте | `meetingplace_news_367`, `meetingplace_news_32`, `regataveka_64` |
| 4 | Как выйти замуж на яхте | `ig_clevel.yacht_DPvo6ACCtm7`, `seapinta_549` |
| 5 | Как люди обычно попадают на свою первую яхту | `silavetrasila_6630`, `silavetrasila_6905`, `silavetrasila_5251` |
| 6 | 5 вещей, которые люди не ожидают от яхтенных путешествий | `LyubimovaEvgeniya_2122`, `LyubimovaEvgeniya_1510`, `LyubimovaEvgeniya_1656` |
| 7 | Что будет на регате в Турции (5 лодок) | `ig_clevel.yacht_DSzdcLSDiVi`, `ig_clevel.yacht_DRe1kpyCjbD`, `silavetrasila_7742` |
| 8 | История: акула и камера | `meetingplace_news_137` |
| 9 | Самые красивые бухты Турции | `ig_anton_timk_DGibmz6v026`, `LyubimovaEvgeniya_386`, `LyubimovaEvgeniya_732` |
| 10 | История: как мы сели на мель | `LyubimovaEvgeniya_2118`, `LyubimovaEvgeniya_1865` |

### Что тестируем

**BM25 (текущий поиск)**
- Baseline: что сейчас возвращается
- Ожидание: хорошо для тем 2, 7, 8 (есть точные слова). Плохо для тем 1, 4, 5 (смысловые запросы)

**Vector search**
- Ожидание: лучше для смысловых запросов (1, 3, 4, 5, 6)
- Риск: может хуже работать для точных имён ("Регата Века", "предпродажа")

**Hybrid search**
- Ожидание: лучший результат по всем темам

### Метрики

- **Hit@5** — есть ли хотя бы 1 релевантный пост в top-5 (0 или 1)
- **Hit@10** — то же для top-10
- **Итоговый score** — сумма Hit@5 по всем 10 темам (максимум 10)

### Как запустить тест вручную

```bash
node --env-file=.env scripts/eval-search.js
```

Скрипт выводит таблицу: для каждой темы — найдено ли попадание и на какой позиции.

### Пример вывода

```
BM25 results:
1. Почему яхтенные путешествия...  ❌ (не нашёл в top-5)
2. До конца предпродажи...         ✅ pos 2
...
Score: 6/10

Vector results:
1. Почему яхтенные путешествия...  ✅ pos 1
2. До конца предпродажи...         ✅ pos 3
...
Score: 9/10
```

## Открытые вопросы

- **Когда обновлять эмбеддинги?** При каждом изменении `textClean` (перескрапинг) — нужно пересчитать. Сейчас посты не редактируются после сохранения, поэтому пересчёт не нужен.
- **Память** — 11k постов × 1536 float32 = ~67MB в RAM при старте поиска. Приемлемо.
- **Скорость** — cosine similarity по 11k векторов занимает ~10мс на Node.js без GPU. Приемлемо.
